# Project Notes

Internal engineering planning and decision records live here.

- `issues/`: problem statements, open questions, and current workarounds
- `specs/`: proposed implementation plans and enhancement designs
- `decisions/`: accepted design decisions and architecture records

Preferred lineage:

1. An issue documents the problem.
2. A spec proposes the implementation.
3. A pull request implements the spec.

When possible, link these documents bidirectionally:

- issue -> related spec
- spec -> addresses issue
- spec -> implemented by PR
- issue -> implemented by PR
